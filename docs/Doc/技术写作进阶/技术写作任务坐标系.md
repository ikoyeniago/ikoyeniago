画这张图的初衷是为了梳理工作过程中各类写作任务的关系，从刚入职时单纯的谁要给谁写，到尝试规范内容开发的流程，到关注技术内容到底如何发挥价值，到随着公司进入下一个阶段产生更多产品、更多职能的内容需求，尝试建立一个多元的体系来支持技术写作相关任务的组织和跟进，可以当做一个浓缩版的需求文档。

当然，ToC、ToB在一些方面会存在差异，但抽象到一定程度，也必然能够找到共性。
